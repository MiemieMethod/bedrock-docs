# 附加包

**附加包（Add-on）**是Minecraft基岩版官方支持的模组格式，用于对游戏内容进行修改和扩展。附加包是基岩版数据驱动生态的核心载体，几乎所有通过官方渠道进行的内容定制都以附加包的形式实现。

## 概述

附加包本质上是一组按照特定目录结构组织的文件集合，通常以ZIP格式打包分发。附加包中的文件包括JSON定义文件、纹理、音效、模型、动画、脚本等，游戏引擎在加载世界时读取这些文件并据此修改或扩展游戏行为。

每个附加包必须包含一个**清单文件（Manifest File）**来声明其基本信息和类型。清单文件的详细说明请参阅[清单文件](manifest.md)文档。

## 附加包类型

附加包根据其包含的内容和作用可分为以下类型：

### 资源包

**资源包（Resource Pack）**负责定义游戏的视觉和听觉表现，属于客户端内容。资源包可以修改或添加纹理、模型、动画、音效、粒子特效、界面等。详见[资源包](resource-pack.md)文档。

### 行为包

**行为包（Behavior Pack）**负责定义游戏的逻辑和行为，属于服务端内容。行为包可以修改或添加实体行为、方块属性、物品定义、配方、战利品表、世界生成规则等。详见[行为包](behavior-pack.md)文档。

### 皮肤包

**皮肤包（Skin Pack）**用于定义玩家的自定义皮肤。详见[皮肤包](skin-pack.md)文档。

### 世界模板

**世界模板（World Template）**是一种预构建的世界，附带有行为包和资源包，用于为玩家提供特定的游戏体验。详见[世界模板](world-template.md)文档。

## 文件扩展名

附加包的打包文件使用以下扩展名：

| 扩展名 | 描述 |
|--------|------|
| `.mcpack` | 单个附加包（资源包、行为包或皮肤包） |
| `.mcaddon` | 附加包集合，包含多个`.mcpack` |
| `.mcworld` | 世界存档，可附带附加包 |
| `.mctemplate` | 世界模板 |

双击这些文件或在文件管理器中打开它们，游戏会自动导入并安装。

## 资源栈

**资源栈（Resource Stack）**是游戏加载附加包时的层次结构。多个附加包可以同时应用于一个世界，它们按照一定的优先级顺序叠加。排列在栈顶的附加包具有最高优先级，可以覆盖排列在其下的附加包中定义的同名资源。

资源栈分为两种作用域：

- **全局资源包**：应用于所有世界的资源包，在游戏设置中管理。
- **世界资源包**：仅应用于特定世界的资源包，在世界设置中管理。

世界资源包的优先级高于全局资源包。

## 依赖

附加包之间可以声明**依赖（Dependency）**关系。例如，一个行为包可以声明它依赖于一个特定的资源包，游戏在加载该行为包时会自动加载所依赖的资源包。依赖关系在清单文件的`dependencies`字段中声明。

## 基包

**基包（Base Pack）**是附加包开发中的一个概念，指当前附加包所继承的基础包。基包提供了默认的资源和行为定义，当前附加包可以选择性地覆盖其中的内容。原版游戏本身即为所有附加包的隐含基包。

## 实验性功能

部分附加包API在正式发布前会以**实验性（Experimental）**状态提供。使用实验性功能的附加包需要在创建世界时手动开启对应的实验性开关。实验性功能可能在后续版本中发生变更或被移除。

- {{file|manifest.json}}：清单文件。
- {{file|pack_icon.png}}：包图标。
- {{file|folder}}**entities/**：实体定义。
- {{file|folder}}**blocks/**：方块定义。
- {{file|folder}}**items/**：物品定义。
- {{file|folder}}**recipes/**：配方定义。
- {{file|folder}}**loot_tables/**：战利品表。
- {{file|folder}}**spawn_rules/**：生成规则。

- {{file|folder}}**scripts/**：脚本文件。
- {{file|folder}}**animation_controllers/**：动画控制器。
- {{file|folder}}**features/**：地物定义。
- {{file|folder}}**feature_rules/**：地物规则。
- {{file|folder}}**biomes/**：生物群系定义。
- {{file|folder}}**dimensions/**：维度定义。
