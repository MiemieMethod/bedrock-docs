# 模组SDK<!-- md:flag china -->

**模组SDK（Mod SDK）**是中国版《我的世界》面向玩法开发提供的一套Python运行时开发框架。该框架以**模组API（Mod API）**为基础接口层，并可扩展到**预设API（Preset API）**与**Apollo网络服（Apollo Network Server）**相关能力，用于构建中国版客户端与服务端联动玩法。

## 定位

模组SDK的核心定位是“运行时逻辑扩展”。其主要职责是：

- 提供系统注册、事件订阅、组件工厂等运行时入口。
- 提供实体、玩家、方块、世界、UI、特效等接口域。
- 支持中国版特有的平台能力，如大厅存储、商城入口、网络服扩展接口。

模组SDK并非国际版`@minecraft/server`脚本API的同名实现。两者在语言、运行时、接口命名和生命周期模型上均不同。

## 组成结构

模组SDK在工程层面通常由以下部分构成：

1. 模组入口与系统类。
2. 服务端与客户端事件监听逻辑。
3. 引擎组件调用与数据同步逻辑。
4. 可选扩展层（预设架构、Apollo网络服、平台能力）。

从文档体系看，模组SDK相关内容可分为“概念、实践、参考”三层：

- 概念层：本页与术语页，解释“是什么”和“为什么”。
- 实践层：教程分区中的模组SDK系列文章。
- 参考层：中国版模组SDK参考分区中的接口、事件、枚举和值域说明。

## 历史文档结构

中国版旧版技术手册通常将相关内容拆分为三个并列部分：

- **ModAPI（Mod API）**：模组SDK的基础接口层，覆盖系统、事件、组件和各类运行时能力。
- **Apollo（Apollo）**：面向网络服集群的服务角色、事件与平台接口体系。
- **PresetAPI（Preset API）**：面向预设架构与对象模型的扩展接口。

这种分法反映的是历史文档编排方式，而不是三个彼此独立的产品。实际工程中，三者常被组合使用：ModAPI负责基础运行时调用，Apollo负责网络服场景，PresetAPI负责预设玩法对象。

若需要保留旧版原文目录脉络，可参阅[中国版API参考归档概述](../../refs/modsdk/china-api-reference-overview.md)。

## 与附加包体系的关系

模组SDK与附加包数据驱动体系可以协同使用，但边界不同：

- 附加包数据驱动侧重声明式配置与资源定义。
- 模组SDK侧重运行时逻辑控制、交互流程与平台接口调用。

在中国版工程中，常见模式是以资源包、行为包提供资源与定义，再通过模组SDK补充运行时逻辑。
