# InnerCore

**InnerCore（InnerCore）**是面向Android版Minecraft基岩版的传统客户端模组框架。其核心目标是为第三方模组提供统一运行层，使开发者能够在客户端侧使用脚本、Java与原生代码扩展玩法与界面能力。

## 定位

InnerCore属于历史社区生态，不属于Mojang当前官方附加包体系。其能力边界、安装方式和运行稳定性受加载器版本、设备系统版本与目标游戏版本共同约束。
InnerCore是客户端模组中的一条历史技术路线，不等同于客户端模组整体。

## 运行结构

InnerCore通常与Horizon启动器和CoreEngine接口协同工作：

1. 启动器负责分发与启动模组包。
2. 模组包提供运行环境与资源组织。
3. CoreEngine向模组暴露主要接口。

## 与附加包的关系

InnerCore模组可与资源包、行为包并存，但两者并非同一技术体系。InnerCore主要面向客户端侧可编程扩展，而附加包主要面向官方数据驱动格式。

## 现状

InnerCore生态在历史上曾广泛用于Android客户端模组开发。当前其主要价值是旧项目维护与生态研究，不宜作为新项目的首选技术路线。
