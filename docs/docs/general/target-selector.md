# 目标选择器

**目标选择器（Target Selector）**是命令系统中用于动态选择玩家或实体的语法结构。目标选择器使命令能够根据位置、类型、标签、分数、名称等条件选中目标，而不必写死具体玩家名或实体唯一标识。

## 概述

目标选择器以`@`开头，后接选择器变量，并可在方括号中附加选择器参数。例如，`@e[type=minecraft:zombie,r=10]`表示选择执行位置10格范围内的僵尸实体。

目标选择器的结果受命令执行上下文影响。命令的执行位置、执行维度、执行者和朝向都可能改变选择结果，尤其在使用距离、坐标和局部坐标相关参数时更为明显。

## 选择器变量

常见选择器变量包括：

| 选择器 | 作用 |
|--------|------|
| `@a` | 所有玩家 |
| `@p` | 最近的玩家 |
| `@r` | 随机玩家 |
| `@e` | 所有实体 |
| `@n` | 最近的实体 |
| `@s` | 当前执行者自身 |
| `@initiator` | 与NPC交互的玩家 |

不同命令参数对目标类型有不同要求。有些参数只接受玩家，有些可以接受任意实体，有些只允许单个目标。

## 选择器参数

**选择器参数（Selector Parameter）**用于缩小或排序目标选择器的结果。常见参数包括`type`、`name`、`tag`、`family`、`scores`、`x`、`y`、`z`、`r`、`rm`、`c`等。

参数可以组合使用。例如，`@e[type=minecraft:sheep,tag=marked,c=1]`会从带有`marked`标签的绵羊中选择一个目标。参数之间通常是逻辑与关系，即目标必须同时满足所有条件。

## 选择结果

目标选择器可能返回零个、一个或多个实体。若命令参数要求单个目标而选择器返回多个目标，命令可能报错或只按规则取其中一个；若选择器没有匹配目标，命令也可能执行失败。

目标选择器的排序和数量限制会影响最终结果。使用`c`限制数量时，应同时理解默认排序规则，必要时使用更明确的筛选条件避免结果不稳定。

## 开发意义

目标选择器是命令系统实现逻辑判断、批量操作和实体管理的基础。它与[记分板](scoreboard.md)、标签、实体族、坐标和执行上下文结合后，可以表达复杂的地图逻辑。
