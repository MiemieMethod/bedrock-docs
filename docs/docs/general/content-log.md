# 内容日志

**内容日志（Content Log）**是Minecraft基岩版中用于记录附加包和游戏内容加载过程中产生的错误、警告和信息的诊断工具。内容日志是附加包开发者排查问题的主要手段之一。

## 概述

当游戏加载附加包中的定义文件时，如果遇到格式错误、缺失引用、不兼容的格式版本或其他问题，引擎会生成相应的内容日志条目。这些条目包含错误的类型、所在文件的路径以及简要的描述信息，帮助开发者快速定位和修复问题。

内容日志并非默认启用，需要在游戏设置中手动开启。

## 日志级别

内容日志中的条目按严重程度与用途分为以下几个级别：

/// define
错误（Error）

- 表示严重问题，通常意味着相关内容无法正确加载或运行。例如，JSON解析失败、必要字段缺失、标识符引用的资源不存在等。

警告（Warning）

- 表示潜在问题，内容可能仍可加载但行为可能不符合预期。例如，使用了已弃用的字段、格式版本不匹配等。

信息（Info）

- 提供一般性的处理记录。该级别通常只写入日志文件，不作为问题提示。

详细（Verbose）

- 提供更细粒度的处理步骤记录，主要用于定位复杂问题时的上下文。该级别通常只写入日志文件。

## 输出位置与查看方式

内容日志可同时输出到图形界面和日志文件。两种输出开关位于游戏设置中的创作者菜单。

### 图形界面

启用内容日志图形界面后，加载世界时出现错误或警告会弹出日志窗口。也可以通过++ctrl+h++打开历史界面查看旧消息并复制到剪贴板。

### 日志文件

内容日志的输出会写入日志目录下的日志文件。内容日志的文件名格式为`ContentLog__<day>__<date>__<time>.txt`，例如`ContentLog__Saturday__2024_May_25__09_58_15_1.txt`。日志文件为纯文本格式，每条记录包含时间戳、日志级别、来源文件路径和详细描述。

常见路径示例如下：

- Windows：`%APPDATA%\Minecraft Bedrock\logs`
- Android：`*root storage location*/games/com.mojang/logs`
- iOS：`*root storage location*/Minecraft/game/com.mojang/logs`

## 内容日志与脚本日志<!-- md:flag vanilla -->

在国际版中，脚本API的运行时错误和`console.log`等输出也会产生日志信息。脚本相关的日志通过单独的脚本日志通道输出，可以在内容日志界面中查看。开启内容日志的同时也会启用脚本错误和警告的显示。
