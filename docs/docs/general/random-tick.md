# 随机刻

**随机刻（Random Tick）**是游戏在已加载区域中随机选择部分方块并触发其随机更新逻辑的机制。随机刻用于驱动许多不需要精确同步的自然变化，例如作物生长、树叶腐烂、草方块蔓延、冰雪变化和铜氧化等。

## 概述

随机刻属于[游戏刻](tick.md#游戏刻)的一部分。每个游戏刻中，游戏会在参与模拟的区域内按一定规则抽取方块位置；若被抽中的方块具有随机刻行为，该方块便执行对应逻辑。由于抽取过程具有随机性，同类方块不会在同一时间整齐变化。

随机刻只会发生在被加载且参与模拟的区域中。超出模拟距离的区块即使被渲染，也不会执行随机刻。

## 随机刻速度

随机刻速度由游戏规则`randomTickSpeed`控制。基岩版默认值为`1`。将其设为`0`会停止随机刻；提高数值会增加随机方块更新频率，使作物、树叶等依赖随机刻的机制更快发生变化。

过高的随机刻速度可能显著增加服务端负载，尤其在包含大量可随机更新方块的世界中更为明显。开发地图或服务器规则时，应避免将该值长期设置为极端数值。

## 与普通刻的区别

普通游戏刻每刻都会推进世界时间、实体逻辑、红石和命令等系统；随机刻只针对被随机抽中的方块位置。随机刻不能保证某个具体方块在指定时间内一定更新，也不能作为精确计时机制使用。

需要确定延迟的方块行为通常使用[计划刻](pending-tick.md)，而不是随机刻。随机刻适合表现自然、分散、概率性的变化。

## 数据驱动

部分方块组件或原版方块行为会响应随机刻。自定义方块是否能够利用随机刻取决于目标版本中公开的数据驱动接口。即使方块能够接收随机刻，也应将其视为概率性更新，而不是稳定的定时回调。

## 与常加载区域

[常加载区域](ticking-area.md)内的区块在没有玩家附近时仍保持模拟，因此其中方块可以继续执行随机刻。需要离线持续运行的农场、自然扩散系统或命令装置常依赖常加载区域维持刻更新。
