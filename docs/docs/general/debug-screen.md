# 调试屏幕<!-- md:flag dev -->

**调试屏幕（Debug Screen）**是Minecraft基岩版开发构建中可用的诊断界面，实时显示游戏运行时的各项技术参数。调试屏幕是开发者和技术型玩家用于分析游戏性能和排查问题的重要工具。

## 概述

调试屏幕在游戏画面上叠加显示大量的实时数据，包括坐标、帧率、内存使用、区块加载状态、实体数量等信息。该功能仅在开发构建（Developer Build）和部分预览版中可用，正式发行版中不包含完整的调试屏幕。

## 显示内容

调试屏幕通常包含以下类别的信息：

### 位置信息

- 玩家的精确坐标（浮点坐标和方块坐标）
- 所在区块坐标
- 朝向角度（偏航角和俯仰角）
- 所在维度和生物群系

### 性能信息

- 每秒帧数（FPS）
- 内存使用量
- 渲染信息（渲染的区块数、绘制调用数等）
- 服务端刻的用时

### 世界信息

- 已加载的区块数量
- 实体总数
- 粒子数量
- 游戏时间和天气状态
- 亮度等级

## 坐标显示

在正式发行版中，玩家可以通过游戏设置中的“显示坐标”选项在屏幕上显示简化的坐标信息（仅包括玩家坐标和朝向），但无法查看完整的调试信息。

## 与内容日志的关系

调试屏幕关注的是运行时的性能和状态数据，而内容日志关注的是附加包加载过程中的错误和警告。两者是互补的诊断工具，分别用于不同阶段和类型的问题排查。
