# 图集

**图集（Atlas）**是将多张二维图像按规则打包并统一索引的一种资源组织机制。在Minecraft基岩版中，图集用于管理方块、物品及部分界面渲染所需的纹理条目，是资源包纹理系统的核心结构之一。

## 核心作用

图集将原本分散的纹理资源合并到统一的渲染页中，主要作用包括：

- 减少渲染阶段的纹理切换频率，降低渲染开销。
- 为方块、物品等内容提供稳定的纹理短名称索引。
- 与多级渐远纹理和填充机制协同，降低远距离采样时的闪烁与溢色。

## 基岩版中的常见图集

| 图集资源名 | 主要用途 | 数据驱动可扩展性 |
|---|---|---|
| `atlas.terrain` | 方块渲染使用的地形图集 | 可通过{{file|terrain_texture.json}}扩展 |
| `atlas.items` | 物品图标与手持渲染使用的物品图集 | 可通过{{file|item_texture.json}}扩展 |
| `atlas.banner` | 旗帜相关纹理 | 主要为硬编码内容 |
| `atlas.shield` | 盾牌旗帜相关纹理 | 主要为硬编码内容 |

## 与附加包文件的关系

图集条目通常由资源包中的图集定义文件声明，开发流程中常见关系如下：

- 使用{{file|terrain_texture.json}}或{{file|item_texture.json}}声明`texture_data`中的纹理短名称。
- 在方块定义、物品定义或其他渲染相关定义中引用该短名称完成纹理挂接。
- 使用[翻书动画](../addon/flipbook-texture.md)时，通过`atlas_tile`等字段将动画帧绑定到图集条目。

## 兼容与维护

图集配置与版本兼容性、渲染质量和包体维护策略直接相关。开发中通常需要关注以下方面：

- 图集层级参数（如`num_mip_levels`与`padding`）对远景质量和纹理边缘伪影的影响。
- 同一纹理短名称下多变种配置时的引用稳定性。
- 旧项目中历史文件（如{{file|textures_list.json}}/{{file|texutres_list.json}}）的保留语境与迁移必要性。
