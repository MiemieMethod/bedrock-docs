# 常加载区域

**常加载区域（Ticking Area）**是Minecraft基岩版中使指定区块在没有玩家附近时仍保持加载和模拟的区域。常加载区域常用于保证命令系统、红石装置、农场或地图逻辑持续运行。

## 概述

默认情况下，游戏只会完整模拟玩家周围一定范围内的区块。玩家离开后，远处区块会卸载或停止模拟，其中的实体、红石、随机刻、计划刻和命令方块都不再继续运行。常加载区域用于打破这一限制，使指定范围在无玩家靠近时仍参与游戏刻更新。

常加载区域通过`/tickingarea`命令创建、列出和移除。创建时可以按矩形范围或圆形范围指定区块区域，也可以指定名称以便后续管理。

## 限制

常加载区域具有数量和规模限制。每个维度最多可以同时存在10个常加载区域，每个常加载区域最多包含100个区块。超出限制时，命令会拒绝创建新的常加载区域。

这些限制用于防止世界长期加载过多区块而造成性能问题。常加载区域中的区块会持续消耗服务端计算资源，因此不应将大面积区域长期保持常加载。

## 作用范围

常加载区域影响的是区块加载和模拟状态。区域内的方块更新、红石、命令方块、函数触发、实体行为、随机刻和计划刻可以继续推进。区域外的区块仍遵循普通的玩家周边加载规则。

常加载区域不等同于玩家存在。某些需要真实玩家参与的机制仍然可能要求玩家在场，例如需要玩家交互、客户端渲染或玩家输入的行为。

## 与地图制作

常加载区域在冒险地图和服务器中经常用于中央控制室、命令方块系统或跨区域机关。将控制逻辑放置在常加载区域内，可以避免玩家远离后命令链停止执行。

在设计时应尽量集中需要持续运行的逻辑，减少常加载区域数量和范围。若逻辑只需在玩家附近运行，使用普通模拟距离往往更加合适。

## 与存档

常加载区域设置属于世界数据的一部分，会随存档保存。复制或发布包含常加载区域的世界时，应检查这些区域是否仍指向正确的位置，避免无意中保留调试区域或多余的后台逻辑。
