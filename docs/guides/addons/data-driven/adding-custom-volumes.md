# 自定义功能域

缺少足够的国际版功能域教程。已确认到的资料只能确认两点：旧的基岩版开发wiki把功能域归入效果相关数据驱动内容，并描述为“在一定区域内的特定效果和功能”；较新的官方鲜艳视觉资料提到，体积迷雾可以利用既有的迷雾和功能域能力，把迷雾属性应用到离散世界区域。除此之外，暂无公开资料提供稳定JSON结构、目录位置和完整示例。

## 目前可以安全采用的做法

如果你的目标是“某个区域内有特殊效果”，优先使用已经有明确文档的方案：

1. 使用`/fog`命令给进入区域的玩家推入迷雾。
2. 使用函数和目标选择器检测区域内玩家。
3. 使用结构、命令方块或脚本API维护区域触发逻辑。
4. 使用生物群系客户端表现控制天空、水色、环境声音和迷雾。

例如，你可以先写一个进入区域时调用的函数：

```mcfunction title="functions/area/enter_mist.mcfunction"
fog @a[x=0,y=60,z=0,dx=30,dy=20,dz=30] push demo:blue_fog demo_mist
```

离开区域时再按你的地图机制对对应玩家执行`fog <player> remove demo_mist`。这个方案不是功能域本身，但能在资料不足时实现相近效果。

## 何时再使用功能域

等你能从目标版本的官方参考、样例包或导出架构中确认以下信息后，再编写真正的功能域：

- 文件应放在哪个目录。
- 根对象和`format_version`。
- 如何声明区域边界。
- 可以触发哪些效果。
- 与迷雾、生物群系和鲜艳视觉的优先级关系。

在这些信息缺失前，不建议编造JSON示例。本站后续取得更完整资料资料后，再把本页改写成正式教程。
