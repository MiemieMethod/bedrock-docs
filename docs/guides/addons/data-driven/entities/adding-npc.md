# 创建NPC

NPC适合制作教程、剧情、商店入口和地图指引。它可以显示对话，也可以在玩家交互时执行命令。官方NPC编辑流程主要在游戏内完成，但你仍然可以把它和函数、结构、世界模板一起使用。

## 准备世界

创建NPC需要：

- 创造模式。
- 已开启作弊。
- 玩家拥有操作员权限。

放置NPC可以使用创造菜单，也可以运行：

```mcfunction
/give @p spawn_egg 1 51
```

右键放置NPC，右键打开编辑界面；放错时可以左键移除。

正式测试玩家交互时，通常需要切换到生存模式或冒险模式，避免创造模式下再次打开编辑界面。

## 设置名称和对话

NPC名称限制为32个字符。对话框适合放短文本；官方资料记录，对话文本超过约307个字符后会超出界面，因此长剧情最好拆成多个交互或使用NPC对话文件。

编辑时可以：

1. 输入NPC名称。
2. 选择外观皮肤。
3. 编写对话文本。
4. 添加命令或按钮。

## 添加命令按钮

在高级设置中添加一条命令，例如：

```mcfunction
give @p gold_ingot
```

开启按钮模式后，为按钮填写显示文本。玩家点击按钮时会执行对应命令。一个NPC可以添加多条命令；命令会按设置顺序运行。

常见触发方式包括：

- 进入对话时运行。
- 点击按钮时运行。
- 退出对话时运行。

## 和函数配合

如果命令较多，不建议全部写在NPC界面中。可以把复杂逻辑放进行为包函数：

```mcfunction title="functions/npc/reward.mcfunction"
give @p gold_ingot 3
say 奖励已发放
```

然后让NPC执行：

```mcfunction
function npc/reward
```

这样后续修改奖励时，只需要改行为包函数，不需要重新逐个编辑世界里的NPC。
