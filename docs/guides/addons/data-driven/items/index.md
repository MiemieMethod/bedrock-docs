# 自定义物品系列教程

自定义物品通常由行为包物品定义和资源包纹理图集共同组成。简单物品只需要图标、堆叠数量和显示名；复杂物品可能还需要附着物模型、可穿戴组件、耐久、食物、工具能力或脚本组件。

## 基本文件关系

/// html | div.treeview
- `demo_BP`
    - `items`
        - `coin.json`
- `demo_RP`
    - `textures`
        - `items`
            - `coin.png`
        - `item_texture.json`
    - `texts`
        - `en_US.lang`
///

行为包的`minecraft:item`定义物品逻辑，资源包的`item_texture.json`把物品图标短名映射到PNG路径。

## 推荐顺序

1. 先创建不会使用、不会穿戴、不会消耗的普通物品。
2. 再加入配方，让它能在生存模式获得。
3. 如果要在手中显示三维模型，再学习附着物。
4. 如果要有复杂交互，再考虑脚本API或自定义组件。

物品比实体和方块更适合作为第一个数据驱动练习，因为它需要的文件少、测试命令简单，并且错误通常集中在图标短名和行为包定义中。
