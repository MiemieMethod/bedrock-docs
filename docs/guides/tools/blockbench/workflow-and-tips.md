# 界面与工作流

Blockbench把模型制作拆成若干界面模式。熟悉这些模式比记住某个按钮更重要，因为大多数问题都来自“正在错误的模式里做正确的事”。

## 四个模式

/// define
编辑模式

- 用于创建立方体、骨骼、定位器和UV映射。轴心点、父子层级、元素位置和模型结构都主要在这里确定。

绘制模式

- 用于创建、导入和编辑纹理。可以直接在三维视图上绘制，也可以在UV面板中处理细节。

动画模式

- 用于给骨骼、粒子、音效和指令添加关键帧。基岩版实体动画最终需要在客户端实体文件和动画控制器中引用。

展示模式

- 主要用于方块或物品格式，设置第三人称、第一人称、头部、地面、物品展示框和GUI等显示槽位。

## 视图与网格

Blockbench的三维视图使用X、Y、Z三个轴描述宽度、高度和长度。基岩版模型制作通常应保持网格对齐，尤其是盒式UV模型。`1`个模型单位通常对应纹理上的`1`个像素单位；如果模型尺寸和纹理像素比例不一致，纹理会出现拉伸、压缩或混合分辨率感。

常用视图操作包括旋转、拖动和缩放。不同键位映射会改变鼠标操作习惯；如果从Blender、Maya或Cinema4D转来，可以在首选项中切换对应键位映射。

## 大纲与骨骼

右侧大纲负责显示模型层级。基岩版实体模型中，组在很多情况下承担骨骼的角色。建议为每个实体保留一个根骨骼，再把身体、头、四肢和装饰件放入清晰的层级中。

父子层级应从身体中心向外建立。移动躯干时，手臂和头部应跟随；移动手臂时，手持装饰件或定位器应跟随。这样制作动画时才能用少量关键帧控制完整结构。

## 轴心点

骨骼会围绕轴心点旋转。门、手臂、轮子、头部和翅膀都需要把轴心点放在真实关节或旋转中心。轴心点放在几何体中心并不总是正确；如果关节位置错误，动画会表现为部件绕错误位置甩动。

可以在编辑模式使用轴心点工具，也可以通过元素面板的数值输入精确设置。对称部件建议先完成一侧，再复制、翻转并检查轴心点是否也被正确镜像。

## 顶点吸附与缝隙修正

顶点吸附可以把一个元素的顶点对齐到另一个元素的顶点，用于修复小缝隙、微小偏移和难以手工对齐的结构。它也可以临时充当测距工具，用来查看两个顶点之间的距离。

如果两个面完全重合或几乎重合，预览中可能出现闪烁。这种现象通常称为Z冲突。处理方法是调整其中一个元素的位置，或轻微膨胀、收缩其中一个元素，避免两个面争夺同一深度。

## 绘制与外部编辑器

Blockbench的绘制模式适合直接处理像素风纹理。画笔、油漆桶、橡皮、取色器、形状工具和复制粘贴工具足以完成许多实体纹理。需要更复杂的像素绘制时，也可以把纹理保存到外部图像编辑器；文件保存后，Blockbench会重新显示更新后的纹理。

绘制时建议保持以下习惯：

- 先铺中间色，再添加阴影和高光。
- 用透明纹理表达薄片和小细节，不要用大量微小立方体堆出所有像素。
- 经常隐藏遮挡元素，检查动画中可能露出的背面和内侧。
- 对对称结构使用镜像绘制，但完成后仍要检查两侧是否需要差异化细节。

## 备份、截图和参考图

Blockbench默认会定期创建备份。误删或崩溃后，可从帮助菜单打开备份文件夹查找最近工程。即使如此，`.bbmodel`源文件仍应手动保存到项目源文件夹中，并与导出的资源包文件分开管理。

右键预览窗口可以添加参考图或蓝图，也可以选择预设视角。截图功能适合制作教程配图、封面和临时预览图；如果图片要进入本站文档或发布材料，应关闭网格和多余辅助线后再导出。
