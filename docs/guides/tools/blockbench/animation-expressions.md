# 动画表达式

Blockbench的动画时间轴支持在若干数值字段中输入表达式。表达式可用于周期运动、速度控制、变量驱动的表情切换和批量数值处理。基岩版动画导出时，这些表达式通常会以Molang形式进入动画JSON；但Blockbench预览支持的内容不一定都能在Minecraft基岩版中以同样方式运行。

## 表达式基础

表达式使用接近Molang的语法，并返回数值。`0`通常表示假，非`0`表示真。三角函数按角度计算，而不是按弧度计算。

```molang
math.sin(query.anim_time * 360) * 10
```

常见用途包括：

- 用`query.anim_time`驱动循环动作。
- 用三元条件表达式按变量切换状态。
- 用`variable.`或`v.`保存模型预览时需要复用的值。
- 在数值滑块中输入`*2`、`+-1`等表达式，对多个选中元素进行批量处理。

/// warning | 区分Blockbench预览和游戏运行
`time`等Blockbench便捷变量可能只适用于工具预览。面向基岩版导出的动画应优先使用基岩版可识别的`query.anim_time`、`query.delta_time`和标准Molang语法。
///

## 关键帧表达式

关键帧的某个轴可以填写固定数值，也可以填写表达式。只有一个关键帧时，整条通道会按表达式持续求值；存在多个关键帧时，Blockbench会先在各关键帧处求值，再按时间插值。

例如，让头部以两秒为周期轻微摆动：

```molang
math.sin(query.anim_time * 180) * 8
```

表达式与平滑或贝塞尔插值并不总是适合混用。需要稳定导出到通用三维格式时，表达式通常会被烘焙为大量普通关键帧。

## 动画时间更新与混合权重

动画属性中的时间更新表达式决定时间轴如何前进。默认逻辑相当于：

```molang
query.anim_time + query.delta_time
```

把增量乘以`2`会使动画以两倍速度播放：

```molang
query.anim_time + query.delta_time * 2
```

混合权重用于缩放整段动画的幅度。`0.5`表示只播放一半幅度，`2`表示放大为两倍幅度。它适合制作呼吸、抖动、情绪强弱等连续变化效果。

## 变量占位符

变量占位符面板可以在不直接修改关键帧的情况下运行表达式。它常用于预览某个变量状态，例如表情、装备开关或部件显隐。

```molang
v.mouth_shape = 2;
v.eye_shape = 1;
```

Blockbench还提供仅用于预览界面的输入函数，例如开关、滑块和瞬时按钮。它们能快速调整变量，但不应被当作基岩版原生Molang函数使用。要让游戏内也能切换变量，应在动画指令关键帧、动画控制器或客户端实体逻辑中写入可导出的Molang表达式。

## 表情与部件显隐

变量驱动的常见做法是为同一部位准备多个骨骼或元素组，再用缩放表达式决定哪一个可见。例如，`mouth_1`只在`v.mouth_shape`为`1`时保持原大小，否则缩放为`0`：

```molang
v.mouth_shape == 1
```

随后可以用指令关键帧在不同时间修改变量：

```molang
v.mouth_shape = 2;
v.eye_left_shape = 1;
v.eye_right_shape = 1;
```

这种方法适合制作面部表情、发光部件开关和替换式装饰件。实际导入游戏时，应确认所有变量初始化和切换表达式都能被基岩版客户端执行。
