# 官方更新日志<!-- md:flag vanilla -->

开发时如果需要核对版本修复内容、热修复影响范围或某项改动是否已经进入正式版，可以直接查官方更新日志。

## 适合用来确认什么

这里的译文目前以基岩版正式版更新日志为主，适合用来确认：

- 某个版本是否修复了命令、附加包或多人相关问题。
- 某个热修复是否只改行为、不改语法。
- 某次更新的开发者相关改动是否已经进入正式版。

## 如何使用这些日志

### 核对行为修复

如果某个命令、附加包特性或多人流程“语法存在但行为异常”，应优先查看对应版本附近的官方更新日志。命令更新时间线更适合判断“何时加入”，而官方日志更适合判断“何时修好”。

### 核对发布通道

开发测试时，应先分清正式版、预览版和测试版。正式面向玩家发布的内容，通常以正式版日志为最终依据；抢先验证新能力时，再结合预览版、测试版说明。

### 核对历史版本

旧项目迁移时，不要笼统写成“旧版日志”或“历史版本日志”就下结论。应先确认目标版本是否真的对应当前要维护的构建，再回看该版本日志中的具体条目。
