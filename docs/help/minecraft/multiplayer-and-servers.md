# 多人游戏与服务器入口

基岩版创作者经常需要在多人环境中测试资源包、行为包、命令、脚本和服务器插件。不同多人入口的功能边界并不相同，选择错误的测试环境可能导致结果无法代表实际发布环境。

## 加入前提

无论使用哪一种多人入口，以下条件都应先满足：

- 所有参与者应使用彼此兼容的游戏版本。正式版、预览版和测试版之间不能混用。
- 设备应登录拥有Minecraft的Microsoft账号；若登录状态异常，可先参阅[账号登录与多人权限](account-sign-in-and-multiplayer.md)。
- 主机世界需要启用多人游戏；加入Realm还需要获得宿主邀请。
- 在主机平台上联机时，主机通常还需要满足对应平台的在线服务订阅要求，例如Xbox Game Pass Core、Nintendo Switch Online或PlayStation Plus。
- 对于儿童账号，还需要在Microsoft或Xbox家庭安全设置中允许多人游戏、跨网络联机和好友功能。

## 局域网世界

局域网世界适合在同一LAN内快速测试多人行为。LAN联机默认指设备位于同一局域网，且不依赖互联网连接。主机需要在世界设置的“多人游戏”页启用多人游戏和对LAN玩家可见，其他设备可在世界列表中看到带有LAN标记的世界。

如果设备已经连接到互联网，游戏可能优先显示在线多人入口；若看不到LAN世界，可检查世界列表顶部的网络切换按钮、同一网络连接状态、防火墙设置、设备是否支持LAN联机，以及游戏版本是否一致。

## 在线服务器

在线服务器适合测试长时间运行的世界、服务器权限、网络延迟和多人规模。计算机和移动设备通常可以在“服务器”页手动添加服务器地址与端口；控制台版本通常只能使用精选服务器入口。这里需要特别区分**专用服务器**与**合作服务器**：前者由个人或社区自行托管，后者是与Minecraft官方合作的精选入口。两者不应混为一谈。

基岩版服务器默认端口为`19132`。客户端与服务器的游戏版本必须兼容；若网络协议版本不一致，客户端通常无法加入服务器。若需要手动添加服务器，应在服务器名称、服务器地址与端口三项信息中至少正确填写地址与端口。

## Realms<!-- md:flag vanilla -->

Realms是官方托管的多人世界服务。它适合测试跨设备加入、资源包自动下发和面向普通玩家的世界体验。加入者不需要拥有Realm订阅，但需要被邀请并满足多人游戏账号设置要求。

Realms不等同于基岩版专用服务器。它不提供插件加载器环境，也不适合测试依赖BDS文件结构或控制台命令的服务器侧功能。

## 基岩版专用服务器

基岩版专用服务器适合测试真实服务器部署、插件加载器、白名单、权限文件、资源包下发和长期运行性能。下载入口位于Minecraft.net的Bedrock服务器下载页；该页面提供Windows和Linux构建，不提供macOS构建，并区分正式版与预览版下载。下载包中还会附带“How to use the dedicated server”说明文件。关于BDS的概念和配置文件，请参阅[基岩版专用服务器](../../docs/server/bds.md)文档。

## Minecraft教育版专用服务器<!-- md:flag edu -->

Minecraft教育版还提供面向教育租户的专用服务器。该服务器与普通BDS相似，能够托管持久世界，但需要教育版客户端、租户管理员配置和教育版专用的服务器注册流程。教育版服务器可设置服务器ID、广播、通行码、允许列表和跨租户相关权限；从1.21.133起，教育版专用服务器还可以按服务器启用聊天日志。

## 常见排查入口

- 登录失败、账号买错或儿童账号权限不足：参阅[账号登录与多人权限](account-sign-in-and-multiplayer.md)。
- 需要确认是否应使用正式版、预览版或测试版：参阅[预览版、测试版与更新日志](preview-beta-changelogs.md)。
- 需要下载官方基岩版专用服务器软件：访问Minecraft.net的Bedrock服务器下载页，或先阅读本页中的专用服务器说明。
