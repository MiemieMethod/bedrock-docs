# 导入与启用内容

Minecraft基岩版可以从文件系统导入世界、附加包、皮肤包和世界模板。导入只会把文件加入游戏的内容存储，是否在某个世界中生效，还取决于世界设置中的资源包和行为包启用状态。

## 文件类型

| 扩展名 | 用途 |
|--------|------|
| `.mcpack` | 导入一个资源包、行为包或皮肤包。 |
| `.mcaddon` | 导入一个附加包集合，通常包含多个`.mcpack`。 |
| `.mcworld` | 导入一个世界存档。 |
| `.mctemplate` | 导入一个世界模板。 |

双击这些文件，或在系统文件管理器中选择使用Minecraft打开，通常会启动游戏并自动导入。导入完成后，游戏会在屏幕顶部显示成功或失败提示。

## 启用附加包

导入资源包或行为包后，需要在目标世界中启用相应内容。

为新世界启用附加包时，可在创建世界流程中进入“资源包”或“行为包”，从“可用”列表中选择目标包并启用。为既有世界启用附加包时，可在世界列表中选择铅笔图标，进入世界设置后在“资源包”或“行为包”中启用。

/// warning | 先备份世界
为既有世界启用行为包、实验性功能或市场内容前，应先导出世界备份。行为包可能改变世界数据、实体状态或方块状态，部分更改在关闭包后也不一定能够完全还原。
///

## Realms中的附加包<!-- md:flag vanilla -->

在Realms中启用附加包时，需要编辑Realm正在使用的世界槽位，并在该世界的资源包或行为包列表中启用内容。变更保存后，游戏会上传修改后的世界设置。其他设备加入该Realm时，会自动下载并应用Realm要求的资源包和行为包。

## 市场附加包与自制附加包

Minecraft市场中的“附加包”通常也是资源包和行为包的组合，但其获取、授权和更新由市场系统管理。自制附加包则通常以`.mcpack`、`.mcaddon`或项目文件夹形式流转。两者在世界中都通过资源包和行为包界面启用，但市场内容可能受到平台、账号和授权状态限制。

关于附加包本身的概念、结构和扩展名，请参阅[附加包](../../docs/addon/addon.md)文档。

如果需要在既有世界中开启实验性玩法，请参阅[开启实验性玩法](enabling-experimental-features.md)。
