# 预览版、测试版与更新日志<!-- md:flag vanilla -->

Minecraft基岩版通过正式版、预览版和测试版分发不同稳定程度的游戏构建。创作者在验证附加包、脚本API、命令或服务器兼容性时，应区分这些构建，并结合官方更新日志判断某项变更是否已经进入正式版本。

## 正式版

正式版是面向普通玩家发布的稳定版本。附加包、服务器和世界模板面向公开分发时，应以正式版作为主要兼容目标。正式版更新日志通常会列出玩法变更、漏洞修复和技术更新，其中“技术更新”“附加包和脚本引擎”“API”“命令”“编辑器”等小节与创作者关系最密切。

## 预览版与测试版

预览版和测试版用于提前测试开发中的功能。

- **预览版**可用于Windows、Xbox、PlayStation和iOS等平台。
- **测试版**主要用于Android平台。

官方帮助中心还明确指出，Beta与Preview除平台差异外，测试内容原则上相同。Windows端安装Preview时，当前帮助中心路径为：打开启动器，选择**Minecraft: Bedrock Edition**，在**Latest release**菜单中切换到**Latest preview**后安装或启动。

这些版本可能包含尚未稳定的功能，也可能出现崩溃、存档损坏或行为变更。用于生产分发的包不应只在预览版或测试版中验证。需要测试新功能时，应使用专门的副本世界，不应直接打开唯一的开发主世界。

### 安装入口

官方帮助中心当前把预览版与测试版的安装说明拆分为多个页面：

- 移动设备上的测试版或预览版安装。
- Windows PC上的Preview安装。
- 主机平台上的Preview安装。

选择入口时，应先确认目标平台是否属于Android测试版通道，或属于其他平台的Preview通道。

### 多人兼容性

多人测试时，所有参与者必须使用同一预览版或测试版构建。若测试Realm，还需要持有有效的Minecraft基岩版Realm订阅。

官方帮助中心同时说明，Beta与Preview中的问题可能超出常规客服支持范围。遇到缺陷时，更适合通过Minecraft Feedback与错误报告渠道反馈，而不是假定客服会为开发中功能提供完整排障。

## 更新日志的使用方式

官方更新日志适合用于回答以下问题：

- 某个方块、物品、生物群系、命令或API是否在某个版本中发生变化。
- 某个已知漏洞是否已经被修复。
- 某个实验性功能是否已经进入正式版。
- 某个预览版变更是否只存在于开发构建中。

阅读更新日志时，应优先确认日志标题中的版本号和分发通道。正式版日志标题通常包含“Bedrock”，预览版或测试版日志标题通常包含“Beta & Preview”。Java版快照日志不应作为基岩版开发结论的依据。

## 与版本概念的关系

更新日志中的外显版本、游戏内部版本、网络协议版本和脚本API模块版本并不总是同一概念。关于这些版本类型的区别，请参阅[版本](../../docs/general/version.md)文档。
