    mod.json 的配置

    这一章深入讲解如何自定义奖池

    警告

    注意，所有配置的读取逻辑都以 服主正确填写格式 为前提

    请认真参考文档中的格式，漏一个空格或字母都有可能导致无法正常运行，且不会有明确的error日志

    mysql_pool_size

    • 含义：功能服的数据库连接池大小，请根据实际需求和性能调整
    • 类型：int
    • 示例
    "mysql_pool_size": 20

    card_pool

    • 含义：奖池列表
    • 类型：list(dict(一个奖池数据))
    • 示例
    {
    ...

    "card_pool": [
        {
            //奖池1数据
        },
        {
            //奖池2数据
        }
    ]
    ...
}

    identifier

    • 含义：奖池的标识符，参照物品/方块标识符规范
    • 类型：str
    • 示例
    "identifier": "hzi:test_pool"

    neteaseMenus

    • 含义：官方菜单插件的联动按钮名称列表
    • 类型：list(str)
    • 示例
    // 启用
"neteaseMenus": ["GachaChest", "Button2"],
// 不启用
"neteaseMenus": null,

    rightTitle

    • 含义：界面右边标题

    • 类型：str

    • 如图

    • 示例
    "rightTitle": "抽 奖"

    leftTitle

    • 含义：界面左边标题

    • 类型：str

    • 如图

    • 示例
    "leftTitle": "奖 池"

    startMsg

    • 含义：开始按钮文案，建议标注消耗货币/物品

    • 类型：str

    • 如图

    • 示例
    "startMsg": "点击抽奖一次, 消耗10"

    greetingsMsg

    • 含义：点击开始抽奖后按钮锁死文案

    • 类型：str

    • 如图

    • 示例
    "greetingsMsg": "天使降临中...q(≧▽≦q)"

    openMsg

    • 含义：打开奖池后调用 neteaseAlert 弹出的提示，如果没有就不会弹

    • 类型：str

    • 如图

    • 示例
    // 弹出
"openMsg": "wow!恭喜发现宝藏!"
// 不弹出
"openMsg": null

    serverType

    • 含义：热更时功能服下发数据的类型列表，建议写上所有实装此奖池的服务器类型，如果此条配置不正确会导致这个运营指令失败

    • 类型：list(str)

    • 示例

    "serverType": ["lobby"]

    enable

    • 含义：启用此奖池，可以用运营指令临时更改此状态

    • 类型：bool

    • 示例

    // 启用
"enable": true
// 不启用
"enable": false

    enableTime

    • 含义：设定此奖池为限时活动奖池，严格按照示例格式输入开始时间到结束时间

    • 类型：list

    • 示例

    // 是限时活动奖池
"enableTime": ["2020-06-08 00:00:00", "2021-06-30 00:00:00"]
// 一直开放
"enableTime": []

    costCurrency

    • 含义：抽奖1次耗费 neteaseTrade 中的货币，可以设置多个，与 costItem 不冲突但优先级比 costItem 高

    • 类型：list(str)

    • 示例

    // 假设抽一次扣除名为RMB的货币10个
"costCurrency": ["RMB*10"]
// 假设抽一次扣除名为RMB的货币10个, 同时扣除USD货币2个
"costCurrency": ["RMB*10", "USD*2"]
// 假设抽此奖池不耗费任何货币
"costCurrency": []

    costItem

    • 含义：抽奖1次耗费背包中的物品，可以设置多个
    • 暂时仅依据newItemName和newAuxValue扣除和回滚（即无法指定扣除玩家的附魔弓，可以指定扣除弓；扣除了附魔弓如果回滚仅会给回弓）
    • 统一使用新版的命名，详见1.23版本物品id变更
    • 扣除数量支持超过最大堆叠数，扣除和回滚时会自动处理
    • 类型：list(str)
    • 示例
    // 假设扣除苹果10个
"costItem": ["minecraft:apple@0*10"],
// 假设抽此奖池不扣除任何物品
"costItem": []

    notEnoughMsg

    • 含义：当检测到玩家的货币/物品不足时弹出提示文案

    • 类型：str

    • 如图

    • 示例
    "notEnoughMsg": "需要10个金币和10个苹果"

    displayFrame

    • 含义：右侧渲染图贴图路径，注意分辨率，参考MC九宫格使用与贴图规范 | MC 开发者文档 (163.com)

    • 类型：str

    • 如图

    • 示例
    "displayFrame": "textures/ui/hzi_gachachest/display/default"

    currencyIcon

    • 含义：按钮旁边的货币图标贴图

    • 类型：str

    • 如图

    • 示例
    "currencyIcon": "textures/ui/hzi_gachachest/currency/currency3"

    winAudioName

    • 含义：抽奖跑马灯动画播放完毕后播放的音效，详情见 sound_definitions.json ，也支持自定义

    • 类型：str

    • 示例

    "winAudioName": "firework.launch"

    selectedAudioName

    • 含义：抽奖跑马灯动画运动时播放的音效，详情见 sound_definitions.json ，也支持自定义

    • 类型：str

    • 示例

    "selectedAudioName": "random.click"

    speedStepSize

    • 含义：动画速度步长

    • 类型：float

    • 示例

    "speedStepSize": 0.1

    maxRoundSpeed

    • 含义：最大动画速度（速度数值越大实际越慢）

    • 类型：float

    • 示例

    // 达到此速度后不会再减速
"maxRoundSpeed": 30.0

    tryStopSpeed

    • 含义：实际开停速度

    • 类型：float

    • 示例

    // 达到此速度后遇到奖品槽位停止动画
"tryStopSpeed": 6.0

    animationStopStayTime

    • 含义：跑马灯动画播放完之后，选框停留多少秒(可以写0就是不等待)

    • 类型：float

    • 示例

    // 两秒之后派发奖品
"animationStopStayTime": 2.0

    animationStopClose

    • 含义：上述等待时间过后是否关闭奖池界面, 不关闭就可以重复抽奖

    • 类型：bool

    • 示例

    // 等待时间过后关闭界面
"animationStopClose": true

    gridDimension

    • 含义：奖池网格规模
    • 不建议改，但不是不可以改，如果25格位置不够用可以尝试扩大，一般需要配合hziGachaChest/resource_packs/hziGachaChestRes/ui/hzi_gachachest_main.json中item_btn@common.button的size参数
    • 类型：float
    • 示例
    // 扩大网格大小
"gridDimension": "5x6"
// 保持默认5x5不动, 节约性能
"gridDimension": null

    centerSlot

    • 含义：中心点的物品槽位，指跑马灯过程中不断变化为选中槽位物品且带附魔的那个槽位

    • 类型：int

    • 如图

    • 示例
    "centerSlot": 13

    centerDefault

    • 含义：还没有点击抽奖的时候中心点槽位展示的物品

    • 类型：dict

    • 格式：参考 物品信息字典

    • 如图

    • 示例
    "centerDefault": {
    "newItemName": "minecraft:lever",
    "newAuxValue": 0
}

    adornmentSlot

    • 含义：装饰槽位列表，指从头到尾都不变的为了好看摆在那的工具人槽位

    • 类型：list(int)

    • 如图

    • 示例
    "adornmentSlot": [7, 8, 9, 12, 14, 17, 18, 19]

    adornmentItem

    • 含义：装饰槽位物品数据，如果数量和 adornmentSlot 一致，那么装饰槽位将按照的顺序依次排布，如果数量不一致，将会从列表里随机抽取物品排列
    • 类型：list(itemDict)
    • 格式：参考 物品信息字典
    • 示例
    // 所有装饰都固定一个物品
"adornmentItem": [
    {
        "newItemName": "minecraft:stained_glass",
        "newAuxValue": 0,
        "count": 1
    }
]
// 每个装饰槽位一半概率是苹果, 一半概率是白色玻璃
"adornmentItem": [
    {
        "newItemName": "minecraft:stained_glass",
        "newAuxValue": 0,
        "count": 1
    },
    {
        "newItemName": "minecraft:apple",
        "newAuxValue": 0,
        "count": 1
    }
]
// 每个槽位都要固定是某个物品, 按照 adornmentSlot 顺序排位
"adornmentItem": [
    {
        // adornmentSlot 中第一个物品
    },
    ...
    {
        // adornmentSlot 中最后一个物品
    }
]

    prizeList

    • 含义：奖池里的奖品列表
    • 类型：list(prize) 列表的顺序即为跑马灯轮换的顺序

    • 格式：奖品信息字典可视为继承了 物品信息字典，在此基础上添加了几个键，这几个键只影响奖池中的显示效果和抽取时的计算，不会影响玩家到手的物品，也不会写入数据库

    • 示例

    "prizeList": [
    {
        // 槽位, 必填
        "slot": 1,
        // 抽取权重, 也就是相对概率, 必填
        "weights": 1,
        // 物品信息面板停留时间(秒), 不填默认0.63秒
        "borderStayTime": 1.0,
        // 物品信息面板文案, 不填默认取此物品的中文名
        // 但不包括附魔等数据, 和customTips不冲突
        "showTips": "&e啦啦啦",
        // 如果要二次开发建议带上, 用于识别特殊物品
        "tag": "hzi:vip_test",
        // 后面就是常规的物品数据
        "newItemName": "minecraft:ink_sac",
        "newAuxValue": 0,
        "count": 1
    },
    ...
  ]
    • 权重的含义

    说人话就是当所有物品权重都相同时，大家均分概率

    假设有两个物品，物品1权重1，物品2权重9，那么抽中物品1的期望概率是10%，物品2是90%

    说实话我也不太懂，建议甩给数值策划
