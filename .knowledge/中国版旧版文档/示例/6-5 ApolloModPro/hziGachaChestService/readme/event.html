    事件

    PlayerGachaEvent

    • 含义：玩家抽奖并检查完库存时触发此事件

    • 适用于：lobby/game

    • 参数：

    名称 类型 说明
    uid int 抽奖的玩家uid
    identifier str 抽的奖池标识符
    enough bool 是否有足够货币和物品, 可以强制修改
    cancel bool 是否取消此次抽奖
    cancelMsg str 取消的理由，例：今日抽奖次数已达上限 留空不提示
    • 示例
    class DevServer(ServerSystem):
    def __init__(self, namespace, systemName):
        ServerSystem.__init__(self, namespace, systemName)
        self.ListenForEvent('HziGachaChest', 'HziGachaChestDev', 'PlayerGachaEvent', self, self.PlayerGachaEvent)

    def PlayerGachaEvent(self, e):
        logging.debug(e)

    PlanningPrizeEvent

    • 含义：计算完玩家即将获得的奖品时触发此事件

    • 适用于：lobby/game

    • 参数：

    名称 类型 说明
    uid int 抽奖的玩家uid
    prizeList list(prize) 奖池的奖品列表, 可以从这里取其他奖品作为参考
    prize dict 奖品itemDict, 可以修改以干预抽奖结果, 要求字典里必须有一个slot以发给客户端展示动画用
    • 示例

    def __init__(self, namespace, systemName):
        ServerSystem.__init__(self, namespace, systemName)
        self.ListenForEvent('HziGachaChest', 'HziGachaChestDev', 'PlanningPrizeEvent', self, self.PlanningPrizeEvent)

    def PlanningPrizeEvent(self, e):
        logging.debug(e)

    PlayerAcquirePrizeEvent

    • 含义：即将派发奖品时触发此事件

    • 适用于：lobby/game

    • 参数：

    名称 类型 说明
    uid int 抽奖的玩家uid
    orderId str 数据库中的订单id
    prizeData dict 奖品itemDict
    cancel int 默认0不取消
    1代表不发放物品，但数据库记录为已发放
    意思是服主自己根据物品tag处理特殊的获奖逻辑
    比如此物品只是用作展示，实际上奖品并不是一个物品而是某种权限
    2代表即不更新数据库状态，也不发放物品
    一般用于特殊逻辑执行上如果失败，可以设置为2再等待补发
    • 示例
    class DevServer(ServerSystem):
    def __init__(self, namespace, systemName):
        ServerSystem.__init__(self, namespace, systemName)
        self.ListenForEvent('HziGachaChest', 'HziGachaChestDev', 'PlayerAcquirePrizeEvent', self, self.PlayerAcquirePrizeEvent)

    def PlayerAcquirePrizeEvent(self, e):
        ###
        假设抽中这个物品就给一个vip权限, 而不是真的把这个物品放到背包
        ###
        def CallBack(suc):
            if suc:
                e['cancel'] = 1
             else:
                e['cancel'] = 2
        if 'tag' in e['prizeData'] and e['prizeData']['tag'] == 'hzi:test_vip':
            self.SetVip(e['uid'], CallBack)
