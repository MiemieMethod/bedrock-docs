插件介绍：
该服务器Mod隶属于“面板描述”插件。
“面板描述”插件实现RPG游戏中常见的物品面板显示功能

插件构成：
目前“面板描述”插件包含以下Mod：
- neteaseAttrs：部署于大厅服或游戏服

使用步骤：
（1）在部署配置中，将neteaseAttrs添加至需要的大厅服或者游戏服的mods列表中

插件api：
（1）展示一个物品面板
函数：Detail(detail, x, y)
参数：
    detail: 信息字典（详见示例）
    x: 屏幕的的横轴坐标值（按需自行获取屏幕大小结合点击位置计算）
    y: 屏幕的的纵轴坐标值（按需自行获取屏幕大小结合点击位置计算）
示例：
    '''
    第一个字段'name'不可以省略
    第二栏预设为品质，格式为单行左右各一个文字框，若需要显示这一栏则对应值必须是长度为2的列表或元组，不需要显示这一栏则不要设置'pinzhi'键
    第三栏预设为物品描述，格式为单行一个文字框，若需要显示这一栏则对应值必须是长度大于1的列表或元组，不需要显示这一栏则不要设置'desc'键，结合实例自行理解
    第四栏预设为耐久，格式为单行左右各一个文字框，若需要显示这一栏则对应值必须是长度为2的列表或元组，不需要显示这一栏则不要设置'naijiu'键
    第五栏预设为属性，格式为单行左右各一个文字框，若需要显示这一栏则对应值必须是长度大于1且成为均是长度为2的元组或列表的的列表或元组，不需要显示这一栏则不要设置'shuxing'键，结合实例自行理解
    第六栏预设为宝石属性，格式为单行左右各一个文字框，若需要显示这一栏则对应值必须是长度大于1且成为均是长度为2的元组或列表的的列表或元组，不需要显示这一栏则不要设置'baoshi'键，结合实例自行理解
    第七栏预设为附魔，格式为单行一个文字框，若需要显示这一栏则对应值必须是长度大于1的列表或元组，不需要显示这一栏则不要设置'fumo'键，结合实例自行理解
    缺省的键值对则自动由其下一行顶替位置，可先于游戏中调试查看效果
    '''
    detail = {
        'name': '§g咖喱棒',
        'pinzhi': ("§g史诗", "§g圣剑"),
        'desc': ["这是第一行文字", "不需要加也不要加反斜杠n换行", "有几行列表里面就有几句话", "这里一共四行"],
        'naijiu': ('§g耐久度', '§g666'),
        'shuxing': [("§g力量", "§g+99999"), ("§g智商", "§g-99999%")],
        'baoshi': [("§g红宝石加的力量", "§g+99999"), ("§g红宝石加的智商", "§g-99999%")],
        'fumo': ["这是第一行文字", "不需要加也不要加反斜杠n换行", "有几行列表里面就有几句话", "这里一共四行"]
    }
    import client.extraClientApi as clientApi
    attrsSystem = clientApi.GetSystem("neteaseAttrs", "neteaseAttrsBeh")
    attrsSystem.Detail(detail, 200, 100)
进阶：
若需要获取原版装备附魔信息文本
则需要在服务端获取物品的信息字典（使用GetPlayerItem接口并传递入参getUserData为True）将其推送到客户端
在客户端处调用GetItemFormattedHoverText接口获取
-参考：
    itemDict = {...}  # 服务端推送下来的物品字典
    import client.extraClientApi as clientApi
    comp = clientApi.CreateComponent('', "Minecraft", "item")
    pretty = comp.GetItemFormattedHoverText(itemDict['itemName'], itemDict['auxValue'], True, itemDict.get('userData')).split('\n')
    enchantData = map(lambda s: '§7{}'.format(s), pretty[2:])
    detail['fumo'] = enchantData  # 这个detail就是上面的信息字典参数

额外说明：
若该插件联动战斗插件，则需要在战斗插件设置某物品格式化规则，更新战斗插件后在behavior_packs下找到与modMain.py同级目录下的fmt.py文件
若不填写该文件或删除该文件则走预设代码显示规则（具体效果请自行查看）
下面简述fmt.py文件填写规则（战斗插件附魔没有效果，则不显示附魔栏）
-示例：
FMT = {
    "minecraft:diamond_sword": {  # 此处对应战斗插件物品配置属性时的名称，如果某种装备不需要定制物品面板显示逻辑则不要出现在FMT字典中，若物品名称出现于FMT字典中但下列键均缺失则正常走预设逻辑显示该装备的物品面板
        "name": "§g大宝剑",  # 定义装备的名称，缺失则代表自动获取，必须完整填写，要么不要出现键'name'
        "part": "§g太刀",  # 定义装备的部位名称，缺失则代表自动获取，必须完整填写，要么不要出现键'part'
        "quality": "§g史诗",  # 定义装备的品质文字，缺失则显示空字符串，必须完整填写，要么不要出现键'part'，若该键与键'part'均缺失，则不显示品质栏
        "desc": ["此刀只应天上有，", "即出入无人之境。"],  # 定义装备的描述文本，若需要显示这一栏则对应值必须是长度大于1的列表或元组，不需要显示这一栏则不要设置'desc'键
        "durability": ["§g久耐", "§g"],  # 定义装备的耐久文字，若需要显示这一栏则对应值必须是长度为2的列表或元组，其中的第一个元素为替换“耐久”的文本，第二个元素为当前物品耐久值的颜色符号，不需要颜色则填空字符串，，不需要显示这一栏则不要设置'durability'键
        "attrs": ["§g", "§g"],  # 定义装备的属性格式，左侧写法为定义属性文本与属性值的颜色（一一对应）并自动获取改装备的属性值与属性文本，若想只显示某些属性，则参考下一个装备该处的配置，不需要显示这一栏则不要设置'attrs'键
        "gem": ["§g", "§g"]  # 宝石同上
    },
    "minecraft:golden_sword": {
        'attrs': [("§g(health)", "§g(health)"), ("§g避闪", "§g可以打字(dodge)都行")]  # 文本均不宜过长，具体显示效果自行调试，一个字符串内括号只能出现一组
    }
}
